    排查流程
    
          
    常见 Docker 场景命令组合。
    
        
    
      
    
      
    
        
    查看本地镜像
    1. 先列出镜像和 digest。
    2. 查看层历史和构建命令。
    3. inspect 获取入口点、环境变量、架构。
    docker image ls
docker image history nginx:latest
docker image inspect nginx:latest
    
        
    拉取工具镜像调试
    1. 拉取 busybox/alpine/netshoot。
    2. 临时容器挂载当前目录或加入网络。
    3. 执行 shell/curl/dig 等工具。
    docker pull nicolaka/netshoot:latest
docker run --rm -it --network app-net nicolaka/netshoot:latest sh
docker run --rm -it -v "$PWD:/work" busybox sh
    
        
    打 tag 并上传镜像
    1. 确认本地镜像存在。
    2. 打成 registry/repo:tag。
    3. 登录 registry 后 push。
    docker image ls
docker image tag app:local registry.example.com/team/app:v1
docker login registry.example.com
docker image push registry.example.com/team/app:v1
    
        
    本地构建并推送
    1. 用 buildx 构建。
    2. 需要多架构时指定 platform。
    3. 用 --push 直接推送或 --load 导入本地。
    docker buildx build -t registry.example.com/team/app:v1 .
docker buildx build --platform linux/amd64,linux/arm64 -t registry.example.com/team/app:v1 --push .
    
        
    容器启动失败
    1. 查看 ps -a 和退出码。
    2. 看日志和 inspect。
    3. 覆盖 entrypoint 进入 shell。
    docker ps -a
docker logs --tail 200 app
docker inspect app
docker run --rm -it --entrypoint sh app:latest
    
        
    容器内排查
    1. exec 进入容器。
    2. 查看进程、端口、环境变量。
    3. 复制文件或导出变更。
    docker exec -it app sh
docker top app
docker port app
docker cp app:/tmp/report.txt ./report.txt
    
        
    网络连通性
    1. 列出 network 和容器归属。
    2. inspect 网络。
    3. 用工具容器加入网络测试。
    docker network ls
docker network inspect app-net
docker run --rm -it --network app-net nicolaka/netshoot curl -v http://service:80
    
        
    磁盘空间清理
    1. 先看 docker system df。
    2. 优先 prune 未使用对象。
    3. 强制删除镜像/volume 前确认影响。
    docker system df -v
docker image prune
docker builder prune
docker system prune --volumes
    
        
    Compose 本地联调
    1. 先 config 验证配置。
    2. up 启动并看日志。
    3. 按 service exec/run。
    docker compose config
docker compose up -d --build
docker compose logs -f web
docker compose exec web sh
